# Smithian Fold Theory of Everything

## Third clean-room reconstruction

This repository is the accessible, science-organized third reconstruction of
Smithian Fold Theory (SFT). It is being built from a clean implementation
environment so that an independent reader can inspect every admitted result from
the self-proven foundation through its generated candidates, forcing,
certificates, controls and, where nature is involved, sealed empirical test.

The reconstruction is motivated by growth and future scale. Earlier corpora,
papers and application repositories remain preserved as scientific provenance;
they are not proof dependencies of this rebuild and are not being withdrawn.

**Current phase:** independent foundation reconstruction. The premise-free root
theorem and structural One are the first model-admitted v3 claims. No result is
admitted merely because it was closed in an earlier corpus.

## Scientific constitution

Every contribution is governed by [CONSTITUTION.md](CONSTITUTION.md). In brief:

- the foundation is the machine-checkable self-proven theorem *there is no
  nothing*;
- the SFT derivational domain admits the One and exact positive rational parts
  of the whole, never semantic zero, negative quantities or irrational values;
- a result must force its assembled form from a generated candidate grammar,
  not only force its ingredients;
- no trained, fitted, selected or target-derived parameter can author a law;
- consensus mathematics and scientific models enter only after an SFT
  derivation, through the correspondence boundary;
- measured targets are structurally excluded from derivation and prediction;
- applications can test laws but cannot select them; and
- anything not closed is marked conditional, empirical, frontier or open.

## One admission engine

Every v3 derivation must execute through the
[single SFT admission engine](docs/ENGINE_AUTHORITY.md). No result enters the
model or census unless the engine accepts its foundation trace, zero-parameter
registration, complete generated enumeration, unique forced survivor, adverse
controls and independent validation. Empirical claims additionally require a
blind prediction seal and separate post-seal measurement against registered
data.

The same engine emits a deterministic rejection receipt when an independent
researcher invalidates any required gate.

See the [current engine implementation status](docs/ENGINE_STATUS.md) for the
implemented gates and the capability-interpreter work required before the first
official empirical derivation.

Natural-science work additionally follows
[EMPIRICAL_METHOD.md](docs/EMPIRICAL_METHOD.md): development observation,
explicit law, preregistration, target-inaccessible execution, sealed output,
post-seal measurement, adverse controls and complete-row preservation.

## Accessible commands

The initial Python scaffold is intentionally small:

```sh
python -m sft status
python -m sft verify-all
```

`verify-all` runs repository validation, the complete unit and end-to-end suite,
the 100% core-engine executable-line coverage gate, and an independent rerun of
every derivation in the ordered census. It is the single reviewer command; its
exact gates are documented in [Complete verification](docs/VERIFICATION.md).
