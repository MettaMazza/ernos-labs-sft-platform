# Repository directory map

## Authority and guidance

- `README.md` - public landing page and knowledge tree.
- `CONSTITUTION.md` - non-negotiable scientific admission rules.
- `AGENTS.md` - operational rules for automated assistants.
- `LANGUAGE_POLICY.md` - accessible Python v3 and host boundary.
- `ROADMAP.md` - dependency-ordered reconstruction programme.
- `CONTRIBUTING.md` - submission process.
- `governance/` - Ernos Labs designation and review gates.

## Scientific source

- `sft/foundation/` - theorem, One, parts, Fold and enforcement.
- `sft/mathematics/` - mathematical foundations grouped by subject.
- `sft/information_science/` - information and coding.
- `sft/computation/` - classical computational science.
- `sft/quantum_computation/` - reversible and quantum computation.
- `sft/physics/` - physical laws.
- `sft/chemistry_materials/` - chemistry and materials.
- `sft/biology/` - biological sciences.
- `sft/earth_environment/` - Earth and environmental sciences.
- `sft/astronomy_cosmology/` - astronomy and cosmology.
- `sft/engineering_translation/` - downstream engineering translations.

## Proof and evidence

- `claims/` - one evidence package per claim.
- `census/` - machine-readable global obligation and status registry.
- `experiments/` - development observations, registrations, sealed predictions,
  measurements and unfavorable controls.
- `generated/` - independent generated certificates.
- `tests/` - implementation, proof, control and integration tests.
- `frontier/` - openly unclosed questions and their admission requirements.

## Boundaries

- `correspondence/` - downstream comparison with historical and consensus work.
- `prior-work-ledger/` - sealed identity of earlier SFT work for post-seal
  reconciliation only.
- `applications/frontier/` - placeholders for later clean application rebuilds.
- `publications/` - current and superseded publication artifacts after explicit
  authorization.
- `tools/` - repository validation and reproducibility tools.
