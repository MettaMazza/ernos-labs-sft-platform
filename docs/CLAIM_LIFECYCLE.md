# Claim lifecycle and admission gates

## Stable identity

Every scientific statement receives a stable identifier before implementation.
The identifier names the branch, subject and sequence without encoding the
desired conclusion.

Example shape:

```text
SFT-MATH-EXACT-001
SFT-COMP-COMPUTABILITY-001
SFT-PHYS-EMPIRICAL-001
```

## Status vocabulary

| Status | Meaning |
|---|---|
| `registered` | Question, dependencies, exclusions and intended evidence are fixed |
| `generated` | Candidate grammar and complete declared domain exist |
| `executed` | The registered computation ran and emitted a source-bound receipt |
| `conditionally_closed` | Unique or complete only inside an explicitly bounded grammar |
| `uniquely_closed` | General certificate establishes the declared unbounded or depth-independent result |
| `empirically_tested` | A sealed consequence has been measured against registered observation |
| `independently_replicated` | A separate implementation reproduces the claim and controls |
| `frontier` | Required closure is known and openly absent |
| `superseded` | A later identified claim package replaces the statement while preserving it |

Statuses are not automatically monotone. Failed controls, source drift or a
broken dependency may return a claim to a prior status with a preserved receipt.

## Required package

Each `claims/<claim-id>/` contains:

- `registration.json` - question, scope, dependencies, exclusions and grammar;
- `STATUS.md` - human-readable status and exact boundary;
- `derivation.py` or a branch-appropriate executable source;
- `candidate_census.json` - complete generated domain identity;
- `elimination_receipt.json` - surviving and rejected forms with reasons;
- `controls.json` - false, tampered and boundary controls;
- `certificate.json` - exact machine result and source hashes; and
- generated independent certificate references where required.

Natural-science claims additionally reference their experiment registration,
sealed prediction and post-seal measurement.

All status transitions from `registered` onward are emitted by the single SFT
admission engine. The census is a projection of accepted engine receipts, not a
manually curated source of scientific authority.

## Admission questions

A reviewer must be able to answer yes to all applicable questions:

1. Does every dependency trace backward to an admitted earlier claim?
2. Was the candidate space generated by a declared rule?
3. Is the grammar boundary explicit?
4. Were all generated alternatives executed?
5. Are minimality and named-shape uniqueness separately checked?
6. Are observational and constitutional relations correctly classified?
7. Is target data absent from derivation and blind execution?
8. Do adverse controls fail as registered?
9. Does an independent certificate recompute the result?
10. Is the claim wording no broader than its evidence?

If any applicable answer is no, the claim is not admitted at that status.
