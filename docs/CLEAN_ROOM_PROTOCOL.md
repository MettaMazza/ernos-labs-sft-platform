# Third clean-room protocol

## Objective

V3 must independently reconstruct the complete SFT corpus in new Python source,
new enumerators and new certificates while preserving the earlier corpora as
post-seal comparison evidence.

## Permitted starting authority

The clean room begins with:

- Maria Smith's self-proven foundational theorem;
- the constraints stated in `CONSTITUTION.md`;
- ordinary host-language and operating-system mechanics, quarantined from SFT
  mathematical meaning; and
- registered empirical observations when a natural-science programme explicitly
  requires observational derivation.

It does not begin with earlier generated implementations, certificates, answer
tables or application-selected laws.

## Separation model

Earlier work is represented in `prior-work-ledger/` by identity, location and
hash. The v3 derivation runner has no dependency path into that directory.

For each legacy claim:

1. register the scientific question without importing the old answer;
2. generate the v3 candidate grammar;
3. execute its complete declared census;
4. derive and seal the v3 result;
5. execute adverse controls and independent certification;
6. only then open the prior-work comparison entry; and
7. record agreement, equivalence, stronger closure, weaker closure or
   discrepancy.

Human participants cannot erase their memory of earlier work. Clean-room
independence is therefore established operationally: no imported code or answer
artifact, independently generated candidates, source-bound execution and
post-seal reconciliation.

## Protected boundaries

Automated checks must reject:

- dependencies from `sft/` into `prior-work-ledger/`;
- dependencies from `sft/` into `correspondence/`;
- dependencies from derivation into `experiments/post_seal_measurements/`;
- dependencies from a fundamental branch into `applications/`; and
- generated certificates that simply embed a registered expected answer.

## Discrepancy rule

A mismatch with an earlier corpus is evidence. It is preserved, registered and
investigated. V3 may change only for a newly demonstrated scientific reason, not
to imitate the previous result.

## Publication boundary

The local repository may be built and tested without publication. Creating a
remote, changing preserved repositories, marking papers superseded in their
public records, pushing code or publishing a DOI requires explicit authorization.
