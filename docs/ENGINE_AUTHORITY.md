# The single SFT admission engine

## Authority

Every v3 derivation, empirical law and publication claim must pass through one
versioned admission engine. Draft work may exist outside the engine, but it is
not part of the SFT model, census or publication authority.

The engine is deliberately fail-closed. A missing certificate, unclassified
dependency, incomplete grammar, failed control, target-access violation or
external-validation failure produces a deterministic rejection receipt and
halts the claim before admission.

## One kernel, evidence-specific validation

All claims pass the same constitutional gates. The final external gate depends
on the claim's evidence class:

- **Formal and computational claims:** an independent implementation must
  recompute the sealed result and adverse controls.
- **Natural-world empirical claims:** the independent implementation gate is
  followed by registered, target-inaccessible prediction, pre-comparison seal,
  separate post-seal measurement, complete-row preservation and data-source
  identity.

This does not exempt mathematics from external validation or exempt empirical
science from derivation. It assigns the appropriate external object to each.

## Engine stages

1. **Registration gate** - one root theorem, no axioms, no free parameters,
   admitted dependencies and classified provenance.
2. **Enumeration gate** - declared generator, boundary, duplicate-free complete
   census and completeness certificate.
3. **Forcing gate** - every candidate decided, exactly one survivor, minimality
   and named-shape uniqueness.
4. **Control gate** - false-premise, tampered-source, tampered-artifact and
   boundary controls all behave as registered.
5. **Independent gate** - a distinct implementation recomputes the sealed
   result from declared inputs.
6. **Empirical gate when applicable** - prediction runtime receives no target or
   score, isolation is recorded, output seals first, evaluator opens data later,
   registered measurements execute and all rows are retained.
7. **Admission gate** - only a fully accepted receipt may enter the census.

The engine evaluates stages in order and does not execute later scientific work
after an earlier stage fails.

## Completeness boundary

The engine cannot turn an author's incomplete generator into a complete grammar
by assertion. It requires a claim-specific completeness certificate and checks
that the emitted domain matches the registered cardinality, contains no
duplicates and receives one decision per candidate. A bounded grammar remains
`conditionally_closed` unless a depth-independent certificate is supplied.

## Blind target boundary

Official empirical runs contain two physically separate phases:

- the prediction phase receives registered inputs, frozen law and target
  identities but no target content or comparison implementation, and executes
  with no filesystem, network, subprocess, clock, environment, dynamic-import
  or foreign-function capability; and
- the measurement phase receives the sealed prediction plus target data only
  after verifying the seal.

One standard-library-only policy applies on macOS, Windows and Linux. The engine
requires a sealed capability certificate recording the interpreter, program,
inputs, denied operations, trace, target absence and comparison-code absence.
Separate target custody records that the target was not released before the
prediction seal. Host platform is metadata and cannot select a law or weaken a
gate.

The initial Python vault API enforces phase order. It does not claim that Python
can sandbox arbitrary host code. Official predictions therefore require the
future capability-closed Fold interpreter; arbitrary Python claim code cannot
receive an official isolation certificate.

## Admission and publication

An accepted engine receipt is necessary for census admission. It does not make
an unregistered branch complete.

A branch paper can be declared ready only when:

- the branch's current-knowledge obligation inventory is frozen;
- every obligation has an accepted receipt at its required evidence status;
- no obligation is unclassified or silently omitted;
- all conditional and empirical boundaries are stated; and
- the branch paper itself passes source, evidence-map and adverse-control checks.

The final Theory of Everything paper can be declared ready only when every
registered branch has a completed branch paper and the global census contains
no unclassified or unclosed frontier obligation inside its frozen current-
knowledge inventory. Work outside that inventory may be named as an external
boundary, but it cannot be counted as a completed branch or admitted law.

## Independent invalidation

The same public engine accepts an SFT claim package from any researcher. A
reproducer can regenerate its candidates, run its controls, replace the
independent validator, or supply a tampered package. The resulting accepted or
rejected receipt identifies the exact gate and artifact in dispute.

This makes SFT claims computationally validatable and invalidatable without
requiring trust in an author, institution or opaque predictor.
