# Registered fourth reconstruction: SFT self-hosting

## Declaration

The third clean-room reconstruction uses Python for accessibility. It is an
intermediate implementation generation, not the final native computational form
of SFT.

A fourth independent clean-room reconstruction will derive and implement:

- SFT-native lexical and syntactic forms;
- exact SFT value and type structures;
- proof-carrying evaluation;
- a compiler or interpreter derived from admitted computational laws;
- a self-hosting runtime;
- deterministic receipt and certificate formats;
- SFT-native package, module and composition principles; and
- a bootstrap chain independently auditable from the One.

## Start gate

V4 may begin only when Maria Smith approves a v3 completion report showing:

- foundational and mathematical dependency closure;
- near-complete declared scientific census with every remaining item classified
  as frontier;
- completed classical and quantum computational foundations;
- stable exact proof and receipt formats;
- a sealed description of the language requirements derived from those laws;
- cross-platform Python reproduction; and
- a frozen v3 comparison corpus.

"Near-complete" is not an intuition. The report must list the total registered
obligations, admitted obligations, conditional obligations and explicit
frontier, with no unclassified row.

## Independence

V4 receives the v3 constitution and sealed scientific claim interfaces. It does
not copy the Python implementation. Its language candidate space, evaluator,
compiler and runtime are generated anew.

After v4 seals each result, it may compare against v3. Discrepancies are
preserved and investigated rather than back-fitted.

## Scientific status

This declaration is a roadmap registration, not a claim that the SFT-native
language has already been derived. Until its own candidate grammar and
certificates exist, the language remains `frontier`.
