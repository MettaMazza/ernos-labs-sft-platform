# Next branch plan: Formal Computation

Status: `planned`; no Formal Computation claim is registered or admitted yet.

Formal Computation is the next dependency branch after the frozen Information
Science inventory. Its purpose is to derive what a computation and a universal
computational model are before Computability, Complexity, Algorithms, Program
Semantics or Quantum Computation use those concepts.

## Proposed dependency-ordered inventory

1. **State and transition** - exact configuration carriers, retained transition
   provenance, initial/terminal forms, deterministic and explicitly branched
   relations, traces and halt conditions.
2. **Formal languages and grammars** - generated alphabets, words, formation
   rules, derivation trees, membership and exact language boundaries without an
   imported language model.
3. **Finite automata** - state, symbol, transition, initial and accepting
   structure with complete run and acceptance traces.
4. **Rewriting systems** - exact pattern occurrence, held replacement position,
   rule identity, successor word and terminating/cyclic/branching traces.
5. **Recursive functions** - structural-One bases, successor, composition and
   recursion over generated finite traces.
6. **Lambda-like calculi** - generated term grammar, held binding identity,
   capture-safe substitution and exact reduction without importing the
   conventional calculus as a premise.
7. **Abstract machines** - exact machine descriptions, configurations,
   transition kernels, tapes/stores where forced and complete executions.
8. **Circuits** - generated node, wire and local transformation syntax;
   feed-forward and cyclic boundaries; exact evaluation traces. Size/depth
   complexity remains downstream.
9. **Operational processes** - execution, terminal states, cycles, branching,
   retained histories and failure-halting conditions.
10. **Composition and decomposition** - exact interfaces, serial and parallel
    process composition, components and reconstruction witnesses.
11. **Computational-model equivalence** - bidirectional generated encodings,
    step/trace simulation, result preservation and declared resource overhead.
12. **Universal computation** - one admitted Fold process interprets every
    generated description inside the frozen machine grammar, with exact
    description, trace and boundary records.

## Existing dependencies available

The branch may cite only model-admitted receipts. Foundation supplies canonical
forms, Fold words, structural One, finite generation and the measurement
boundary. Mathematics supplies collections, relations, maps, graphs, algebra,
order, dynamics, proof and composition. Information Science supplies symbols,
encoding, distinction ledgers, channels, loss records and exact classical /
probabilistic / quantum-support boundaries.

## Work required for each obligation

Each claim must be preregistered before execution and must provide:

- a complete exact grammar and explicit boundary;
- generated structural dimensions with rejected and preserving coordinates;
- a complete candidate census and one decision for every candidate;
- exactly one all-preserving survivor;
- minimality and named-shape uniqueness;
- live operational witnesses and deliberately unfavorable controls;
- a structural-One base and successor certificate for generated finite depth;
- empty axiom and free-parameter tuples;
- source-manifest and derivation seals;
- an implementation-distinct validator that regenerates the literal product;
- a model-admitted engine receipt; and
- exact limitations separating Formal Computation from downstream
  Computability, Complexity, Algorithms, Program Semantics and Quantum
  Computation.

## Branch completion gate

The inventory can be frozen only when all twelve obligations are classified and
model-admitted, with no unclassified or hidden frontier row. Only then will a
new standalone Formal Computation paper be written. That paper will reproduce
every derivation chain in full and will not edit the Foundation, Mathematics or
Information Science papers.

No application experiment, prior SFT implementation, conventional Turing
machine, programming language or pretrained model may select these laws.
Historical computational models may enter only after sealing as explicit
correspondence tests.
